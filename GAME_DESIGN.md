# 백팩스위퍼 (Backpacksweeper) — 완성형 게임 기획서 v2.0
### *"가방은 갑옷도 환전소도 아니다 — 가방은 너의 추리를 더 깊게 만드는 '두 번째 보드'다"*

> **본 v2.0은 v1 통합안에 적대적 비평의 8개 designHole·7개 mustFix를 전부 반영해 재설계한 개발 착수용 최종본이다.** v1의 핵심 용접점이었던 "환율 펌프(보상만 증폭)"가 논리적으로 붕괴(EXP↑ = 레벨업↑ = 생존↑)함을 인정하고, **생존 자원과 보상 자원을 코드 레벨에서 물리적으로 분리**하는 새 모델로 교체했다. 모든 변경점은 각 절 끝 `[비평 반영]` 박스에 명시한다.

## 1. 한 줄 피치 + 장르 정의

### 한 줄 피치
> **드래곤스위퍼의 "숫자 = 인접 몬스터 레벨의 총합" 추리를 100% 보존한 채, 백팩히어로의 비정형 가방을 "위험을 줄이는 갑옷"이 아니라 "추리 해상도를 사는 작업대"로 재해석한 격자 추리 로그라이크.**

### 장르 정의
**Deductive Grid Roguelike (연역형 격자 로그라이크).**
- **상위 장르:** 턴리스(turnless) 두뇌 퍼즐 + 로그라이트 메타 진행.
- **핵심 동사:** `읽다(Read) → 추론하다(Deduce) → 지불하다(Pay) → 정비하다(Pack)`.
- **장르적 입장:** 액션 로그라이크가 아니다. 반사신경·DPS·회피가 없다. 모든 위험은 "계산으로 풀리는 정보"이며, 사망은 오직 **계산 실수**에서만 온다. 가방은 전투력이 아니라 **인지 부하를 거래하는 수단**이다.

### 비전 한 문단
플레이어는 매 클릭마다 "이 칸에 무엇이 있는가"를 인접 숫자들의 연립방정식으로 풀어낸다. HP는 체력바가 아니라 **공격에 지불하는 연료**이며, 지불한 만큼 성장(레벨업)과 보상(골드·전리품)이 돌아온다. 가방은 이 추리를 **자동화하지 않는다** — 대신 "어느 칸을 더 선명하게 볼지", "어느 줄에 정비 도구를 정렬할지"를 플레이어가 가방 공간에서 직접 설계하게 하여, **보드 추리와 가방 배치가 하나의 통합된 퍼즐**이 되게 한다.

## 2. 드래곤스위퍼 원작 핵심 정리 (정확성 교정)

> 본 절은 검증된 원작 사실을 기준으로, 흔한 오해를 명시적으로 바로잡는다. **이 절의 모든 수치·규칙은 우리 게임의 추리 무결성 기반이므로 개발 시 신성불가침이다.**

### 2.1 결정적 핵심 — 숫자의 의미 (가장 흔한 오해 교정)

| 오해 (틀림) | 사실 (맞음) |
|---|---|
| ❌ 타일 숫자 = 주변 몬스터 **마리 수** (클래식 지뢰찾기) | ✅ 타일 숫자 = 주변 8칸에 있는 **모든 몬스터 레벨(=데미지)의 총합** |

**예시:** 인접 8칸에 Lv2 슬라임 2마리 + Lv1 박쥐 1마리 → 그 칸 숫자는 **5** (2+2+1).
따라서 숫자 `7`은 다음 중 무엇이든 될 수 있다:
- Lv7 몬스터 1마리 (단, 도감에 Lv7이 존재할 때만)
- Lv5 + Lv2
- Lv4 + Lv2 + Lv1
- ... 등 **도감에 실재하는 레벨들의 조합**

이것이 클래식 지뢰찾기와의 본질적 차이이자 원작의 최대 혁신이다. 몬스터 레벨이 **고정**이므로, 도감 지식 + 인접 숫자들을 **연립방정식**처럼 풀어 각 칸의 정체를 좁힌다.

### 2.2 HP 경제 — "체력이 아니라 소비 자원"

- 시작 HP: **5 하트.**
- 몬스터 클릭(=공격) → 그 몬스터 레벨만큼 HP **즉시** 차감 → **동량의 골드/EXP 회수.**
- **핵심:** 죽지 않는 한 쓴 HP는 EXP로 100% 되돌아온다. 최대 HP는 상한이 있으나 **EXP는 무한 누적.**
- **벽(wall):** HP를 지불해 부수면 골드만 줌 (**EXP는 안 줌 — 유일한 예외**).
- HP 0 = 사망(게임 오버).
- **최적 플레이:** 레벨업/회복 직전에 HP를 거의 0까지 쥐어짜 쓰는 것.

### 2.3 레벨업 = 회복 + 성장의 통합 메커니즘

- EXP를 일정량 모으면 레벨업.
- 레벨업 시 **(1) HP 최대치까지 완전 회복 + (2) 최대 HP +1.**
- 초반 레벨업 비용 약 **4 EXP**, 레벨이 오를수록 증가(대략 단계마다 +3).
- **회복 수단으로 하트 두루마리보다 레벨업이 유리** — 회복+성장을 동시에 주기 때문.
- 목표: **레벨 15 (= 하트 15) 도달** → 중앙의 **Lv15 드래곤** 처치.

### 2.4 도감(Monsternomicon) = 추리 엔진

단순 장식이 아니라 **추리의 필수 도구.** 각 몬스터의:
- **고정 레벨** (= 데미지 = 골드/EXP)
- **외형**
- **결정적 배치 규칙** (추가 단서)

## 3. 코어 자원 순환 루프 (HP 경제) — 비평 #1 핵심 해결

> **이것이 v2.0의 심장이자 v1 대비 가장 큰 변경이다.** 비평이 정확히 지적했다: v1의 "환율 펌프(보상만 증폭, 위험 불변)"는 거짓이다 — EXP를 증폭하면 레벨업이 빨라지고, 레벨업 = 회복 + 최대HP이므로 곧 **생존을 증폭**한다. 따라서 우리는 **HP 지불이 만들어내는 회수를 두 개의 물리적으로 분리된 트랙으로 쪼갠다.**

### 3.1 이중 트랙 모델 (Dual-Track Recovery Model)

몬스터를 클릭해 `Lv = L` 의 데미지를 지불하면, 회수가 **두 갈래로 동시에** 발생한다:

```
                       ┌──────────────────────────────────────────┐
   클릭(=공격)          │  HP를 L 만큼 즉시 차감 (위험 = 고정·불변)   │
   몬스터 Lv = L  ──────┤  ★ 가방은 이 차감액에 절대 개입 못 함 ★     │
                       └──────────────────────────────────────────┘
                                        │
              ┌─────────────────────────┴─────────────────────────┐
              ▼                                                    ▼
   ┌───────────────────────────┐                  ┌───────────────────────────────┐
   │  트랙 A — 생명력 (Vitality) │                  │  트랙 B — 전리품 (Loot)         │
   │  = 지불한 L 그대로 (1:1)    │                  │  = 골드 + "초과 EXP"             │
   │  ★ 가방 개입 불가 (read-only)│                  │  ★ 가방이 증폭하는 유일한 대상   │
   └───────────────────────────┘                  └───────────────────────────────┘
              │                                                    │
              ▼                                                    ▼
   ┌───────────────────────────┐                  ┌───────────────────────────────┐
   │ Vitality 누적 → 레벨업      │                  │ Loot(골드)는 회복·최대HP·레벨업과 │
   │ 레벨업 = 완전회복+최대HP+1   │                  │ 100% 무관. 캠프에서 가방 조형·    │
   │ ★ 순수하게 "지불한 L의 합"에 │                  │ 아이템 구매·해금에만 사용.        │
   │   비례 — 빌드 무관·결정론적  │                  │ "초과 EXP"는 칭호·랭킹·메타 통화로.│
   └───────────────────────────┘                  └───────────────────────────────┘
```

**핵심 분리 원칙 (수치적 증명):**

1. **트랙 A (Vitality / 생존):** 레벨업에 필요한 것은 오직 **`누적 Vitality`**이며, `Vitality += L` (지불한 레벨 그대로, 항상 1:1, 무한 증폭 불가). 두 플레이어가 동일한 몬스터들을 처치하면 **빌드와 무관하게 정확히 동일한 시점에 레벨업**한다.
   - 레벨업 비용: `cost(n) = 4 + 3·(n-1)` Vitality (원작 곡선 채택, JSON 튜닝).
   - → **"같은 HP를 태우면 누구나 같은 속도로 풀피·성장한다."** 환율 빌드가 생존을 가속하는 경로가 **코드 레벨에서 차단**됨.

2. **트랙 B (Loot / 보상):** 골드와 "초과 EXP(점수)"만 가방이 증폭한다. 그리고 **골드·점수는 회복·최대HP·레벨업 비용에 일절 쓰이지 않는다.**
   - 골드 용도: **캠프에서만** — 가방 칸 조형 비용, 아이템 구매/리롤, 메타 해금.
   - 점수(초과 EXP) 용도: 클리어 랭킹, 빌드 태그, 메타 통화(드래곤 비늘) 환산.
   - → **"가방이 폭발시키는 것은 점수판과 다음 판의 선택지이지, 이번 판의 생존이 아니다."**

3. **단일 통화 헌법과의 양립:** 플레이어가 지불하는 통화는 **여전히 HP 단 하나**다. 트랙 A/B는 "지불 후 회수가 두 장부로 기록"되는 것일 뿐, 플레이어가 관리하는 입력 자원은 HP 하나로 유지된다. → 비평 #1의 "(a)안은 단일통화 헌법과 충돌" 우려를 **회피**한다 (이중 *지불* 통화가 아니라 이중 *회수* 장부이므로).

> **[비평 반영 — designHole #1 / mustFix 최우선 #1 해결]**
> v1: `HP 지불 → EXP 회수 → (가방 증폭) → 레벨업 → 회복+최대HP`. 가방이 EXP를 증폭하면 생존이 증폭됨 = 헌법 위반.
> v2: `HP 지불 → [트랙 A: Vitality(1:1, 가방 불가) → 레벨업/생존] + [트랙 B: 골드·점수(가방 증폭) → 캠프 경제/랭킹]`. 생존은 지불한 레벨 합에만 비례하므로 **빌드 무관**. 가방은 생존 경로에 닿지 못한다. 분리가 **수치적으로 증명됨**: 동일 처치 → 동일 Vitality → 동일 레벨업 타이밍.

### 3.2 코어 루프 5단계 (다이어그램식)

```
①  읽기 (READ)        빈 타일 숫자(=인접 8칸 레벨 총합)를 읽는다. 도감 + 가방 정보
                      도구로 각 칸의 정체 후보를 좁힌다. (가방은 read-only)
                            │
②  추론 (DEDUCE)       인접 숫자들을 연립방정식으로 풀어 "안전 칸 / 감당 가능한
                      몬스터 / 위험 칸"을 판정. ★ 여기가 게임의 본질 ★
                            │
③  지불 (PAY)          감당 가능한 칸 클릭 → HP를 L만큼 즉시 차감(모션 0프레임).
                      화면 흔들림 + 빨간 -L 숫자 트윈 + 초록 보상 숫자가 가방으로.
                            │
                ┌───────────┴───────────┐
                ▼                        ▼
        Vitality += L              Loot += (가방 증폭된 골드·점수)
        (생존 트랙, 1:1)            (보상 트랙, 가방이 폭증시킴)
                │                        │
                ▼                        │
④  성장 (GROW)         Vitality가 cost(n) 도달 → 레벨업 → 화이트플래시 →
                      완전회복 + 최대HP+1. ★ 보상 트랙과 무관하게 발동 ★
                            │
⑤  정비 (CAMP)         구역(zone) 클리어 또는 레벨업 시 캠프 진입 → 골드로 가방
                      조형·아이템 구매·재배치. ★ 보드 위에선 절대 불가 ★ (§6.4)
                            │
                      ┌─────┘
                      └──→ 다시 ①로. 최종 Lv15 + 완전회복 → 드래곤 처치 → 승리.
```

### 3.3 "쥐어짜기"가 여전히 최적이지만 강제되지 않는 이유 (비평 #6 선반영)

원작의 "HP를 0까지 쥐어짜기"는 **트랙 A에서 자연스럽게 유지**된다 (레벨업 전 HP를 많이 쓸수록 손실 EXP 없음). 단, v1의 "탐욕의 인장 / 쥐어짜기 게이지"가 만든 **"HP 1 상주 지배전략 → 데스 스파이럴"** 문제는 §7.2에서 보너스 산식을 "HP 비율"에서 "누적 소모량"으로 바꿔 제거한다. → **신중한 계획 플레이와 공격적 플레이가 동등하게 보상받음.**

### 4.2 전투 중 편집 허용 여부 — **확정: 전면 불가 (read-only)** + 비평 #4 충돌 해소

> 비평 #4가 정확히 지적했다: v1은 **1장("레벨업 시 가방 조형 가능")과 6장("캠프에서만 조형")이 정면 충돌**했고, 단일 보드에서 캠프가 1회뿐이면 "패킹=추리" 빌드 깊이를 한 판에 검증 불가능했다. v2는 다음과 같이 **확정 통일**한다:

**캠프(편집 가능 구간)가 열리는 시점 — 3가지 (확정):**

1. **레벨업 순간 = 미니 캠프 (Quick Camp).** 레벨업하면 화이트플래시 후 **인라인 미니 패널**이 열린다. 여기서 **이번에 얻은 조형 칸 1~2개 배치 + 보유 아이템 재배치 + (골드 있으면) 즉석 구매**가 가능하다. 보드는 일시정지(턴리스이므로 시간 압박 없음). → **이것이 "1장의 레벨업 조형"과 "6장의 캠프 조형"을 통일하는 다리.**
2. **구역(Zone) 클리어 = 풀 캠프 (Full Camp).** 단일 보드를 **3개 구역**으로 분할(§8.1). 한 구역의 모든 안전 처치가 끝나면 풀 캠프 진입 → 상점·리롤·대규모 재배치.
3. **드래곤 직전 = 최종 캠프.** 드래곤 처치 전 마지막 정비.

**보드 위(읽기/추론/지불 중):** 가방은 **완전 read-only.** 효과는 전부 패시브 상시 표시. 회전·이동·구매 일절 불가. → 즉발 템포 보존.

> **[비평 반영 — designHole #4 / mustFix #6 해결]**
> v1: 1장과 6장 충돌 + 단일보드 캠프 1회 → 패킹 검증 불가.
> v2: "레벨업=미니캠프" 규칙으로 두 장 통일. 단일 보드를 3구역으로 쪼개 **한 판에 레벨업(다회) + 구역 클리어(3회) = 최소 6~10회의 편집 기회** 보장 → "패킹=추리" 빌드 깊이를 한 판 안에서 충분히 검증 가능.

### 4.3 가방의 격리 규칙 (헌법 — 추리 무결성 방어)

1. 가방은 보드의 **숨은 정보에 직접 작용하지 않는다.** 오직 "플레이어가 가진 정보의 해상도(트랙 정보)"와 "트랙 B 보상 환율"만 바꾼다.
2. 가방 효과는 **전부 패시브·상시 표시.** 플레이어는 가방을 *조작하지 않고 읽고* 클릭한다.
3. **백팩히어로식 좌상→우하 자동 발동(발동 순서 결정론)을 보드에 절대 적용하지 않는다.** 가방 효과는 보드 상태를 자동 변경하지 않는다 (공정성 붕괴 방지).
4. 재배치는 **캠프(§4.2의 3시점)에서만.**

### 4.4 시너지 축 = 단 2개 고정 (헌법 제2조)

> 곱연산·체인 트리거를 도입하면 밸런싱이 붕괴한다(3심사위원 만장일치). 모든 시너지는 **덧셈 또는 1:N 선형 분배까지만.**

- **축 A — 행/열 직선 정렬 (Line):** 같은 행·열에 정렬된 동종 아이템이 서로를 **덧셈** 강화.
- **축 B — 인접/빈칸 (Adjacency / Void):** 인접 아이템 수, 또는 *비워둔 칸* 수에 비례해 **선형** 발동.

**금지:** 체인 트리거, 곱연산, 지수 폭발, 조건부 연쇄.

### 5.6 시그니처 빌드 격자 배치 검증 (비평 #5 / mustFix #7 해결)

> 비평 #5: v1 시그니처 모먼트의 "저울+루비+숫돌3개 한 열(5칸) + 정보 도구"가 4x4(16칸)에 물리적으로 안 들어간다. v2는 **5x5 기반 13칸 시작**으로 상향하고, **실제 배치도를 첨부**한다.

**시작 가방 (13칸 활성, `■`=활성/`·`=비활성/`▣`=돌출슬롯):**
```
     col0 col1 col2 col3 col4
row0  ·    ■    ■    ■    ·
row1  ■    ■    ■    ■    ■
row2  ■    ■    ■    ■    ■
row3  ·    ■    ■    ■    ·
row4  ·    ·    ▣    ·    ·     ← 돌출 격리 슬롯
```

**시그니처 "경제 빌드" 배치 예시 (레벨업 조형 +2칸 가정, col4 한 열에 경제 정렬):**
```
     col0 col1 col2 col3 col4
row0  ·   [측량][측량][측량]  ·      ← 측량 잣대 1x3 (행 정보)
row1 [눈] [숫돌][분렌][프리]  [저울]  ← 분해렌즈(기본)+숫돌+프리즘 / 저울 머리
row2 [눈] [공명][공허][공허]  [저울]  ← 가저의눈 1x3 세로(열 정보) / 저울 1x2 꼬리
row3  ·   [화살][계량][계량]  ·      ← 공허 화살집 꼬리 + 계량추
row4  ·    ·   [저주]  ·     ·      ← 피의굶주림 돌출 단독 격리
```
- **교차점:** `측량 잣대(row0)` × `가저의 눈(col0)` → **(row0, col0) 영역 + 측량행/가저열 교차 칸**이 최고 해상도.
- **경제 한 열:** `상인의 저울(col4)`이 col4 처치 골드 +25%, `숫돌`이 같은 행 점수 가산.
- **검증 결과:** 13칸+조형 2칸 = 15칸으로 **정보 도구(측량+가저+분해렌즈+프리즘) + 무기(저울+숫돌+공허+계량) + 저주 격리**가 **동시 성립.** v1의 4x4에선 불가능했던 빌드가 v2 5x5-13칸에서 물리적으로 들어감을 확인.

> **[비평 반영 — designHole #5 / mustFix #7 해결]** 시작 가방을 4x4→5x5(13칸 활성)로 상향, 실제 배치도 2종 첨부로 시그니처 빌드 성립 입증.

### 5.7 "약칸 연타 노가다 < 강칸 정확 추리" 증명 (비평 degenerate #2 / mustFix #5 해결)

> 비평: 환율 보너스가 "클릭 횟수"에 비례하면, 안전한 약칸(Lv1)을 다량 연타하는 노가다가 강칸 정확 추리보다 효율적 → 사용자가 가장 우려한 "추리 우회 노가다"로 전락.

**해결 — 모든 트랙 B 보너스를 "지불 HP 총량" 또는 "처치 Lv"에 비례시키고, "클릭 횟수"에는 절대 비례시키지 않는다:**

| 산식 유형 | v1 (위험) | v2 (안전) |
|---|---|---|
| 숫돌의 반지 | 처치당 +N 점수 (클릭 횟수 비례 → 약칸 연타 유리) | **지불 Lv당 +N 점수** (`+N×L`) → Lv1 10번 = Lv10 1번, **노가다 무이득** |
| 격노의 루비 (→삭제) | 인접 수만큼 처치당 분배 | **삭제.** 계량추로 대체 |
| 계량추 (신규) | — | `+5%×L` 골드 → **Lv 높을수록 유리** → 강칸 추리 직접 보상 |

**정량 증명 (숫돌 1개, +1점수/Lv 기준):**
- 약칸 연타: Lv1 × 10회 = HP 10 지불, 점수 = 1×10 = **10점**, 클릭 10회.
- 강칸 정확: Lv8 + Lv1×2 = HP 10 지불, 점수 = 8×1 + 1×1×2 = **10점**, 클릭 3회.
- → **동일 HP 지불이면 점수 동일.** 클릭 횟수가 적은 강칸 쪽이 *시간 효율*에서 오히려 우세하므로, **노가다 유인이 0.** 게다가 강칸 처치는 더 어려운 추리를 요구하므로 "추리 실력 → 보상"이 직결.

> **[비평 반영 — degenerate #2 / mustFix #5 해결]** 모든 트랙 B 보너스를 "지불 HP/Lv"에 묶고 "클릭 횟수"와 분리. 약칸 연타 노가다가 강칸 추리 대비 **수치적으로 우월할 수 없음**을 증명.

## 6. 추리 시스템 & 무결성 보장

### 6.1 추리의 본질 (보존)

타일 숫자 = 인접 8칸 몬스터 레벨의 총합. 도감의 **고정 레벨 + 배치 규칙**을 인접 숫자들과 연립해 각 칸의 정체를 좁힌다. **사망은 오직 계산 실수에서만.**

### 6.2 무결성 4대 방어선 (헌법)

1. **어떤 아이템도 답을 확정 공개하지 않는다.** 유물은 "기여 칸 수"나 "배치 규칙"만 보조; 정체는 플레이어가 연립.
2. **위험 흡수·회복·부활 전면 부재** (헌법). "빌드 강하면 깡으로 민다"가 원천 차단. 더해 **트랙 A 분리**(§3)로 "환율 빌드가 생존을 가속"하는 우회로까지 차단.
3. **보드는 항상 "인간 가독성 solvable" 시드** (§6.4에서 강화). 운 사망은 계산 실수에서만.
4. **발동순서 결정론 함정 회피.** 가방은 보드 상태를 자동 변경하지 않음.

### 6.3 분해 렌즈 재설계 — 비평 #2 핵심 해결 (정보이득 정량 검증)

> 비평 #2가 가장 날카롭다: 원작 레벨이 **이산 집합 `{0,1,4,5,8,11,15,100}`**이므로, "7 → 5+2/6+1/4+2+1" 식 분해 표시는 도감 필터와 결합하면 **추리의 가장 무거운 부분(분해 열거)을 자동 대행**한다. 예: 7은 도감에 6·3·2가 없으니 자동으로 후보가 급감 → 렌즈가 연립 1단계를 대신 풀어줌.

**v2 재설계 — 분해 렌즈는 "분해 후보"를 절대 보여주지 않는다. 대신 정보량을 한 단계 낮춘 "기여 칸 수 상한/하한"만 표시:**

| 레벨 단계 | 분해 렌즈가 보여주는 것 | 정보이득 |
|---|---|---|
| **기본 (분해 렌즈)** | 그 숫자에 기여하는 **인접 몬스터 칸의 개수 범위** (예: 7 → "2~4칸이 몬스터") | 낮음 — "몇 칸이 비었나"만. 어떤 레벨인지·어느 칸인지는 미공개 |
| **+측량잣대/가저의눈 (행/열)** | 해당 행/열 교차에서 **정확한 기여 칸 수** (예: "정확히 3칸") | 중간 — 칸 수 확정. 단 *어느* 칸·*무슨* 레벨은 여전히 플레이어가 연립 |
| **교차점 (행×열 둘 다)** | 교차 칸 1개에 한해 **"이 칸이 기여 칸인지 빈칸인지"** 이진 표시 | 높음(국소) — 단 한 칸. 보드 전체로 확장 불가 |

**핵심:** 어떤 단계에서도 **"레벨 분해 후보"를 나열하지 않는다.** 플레이어는 항상 "칸 수 + 도감 레벨 집합 + 인접 단서"로 직접 연립해야 한다. → 비평이 우려한 "분해 열거 자동화"가 **제거됨.**

**MVP 0순위 검증 (mustFix #2 — 코드 1줄 전 스프레드시트 프로토타입):**

> 비평 명령: "정보이득 측정 하베스를 코드 쓰기 전 페이퍼/스프레드시트로 먼저 돌려 분해 렌즈가 추리를 자동화하는지 판정하라."

**검증 절차 (개발 착수 전 필수, 산출물 = 스프레드시트):**
1. 대표 시드 20개를 손으로 생성.
2. 각 칸에 대해 "렌즈 없음 / 칸수 상하한만 / 정확 칸수 / 교차점 이진" 4조건에서, **남은 가능 정체 후보 수**를 계산.
3. 지표: `평균 후보 감소율 = (렌즈 전 후보수 − 렌즈 후 후보수) / 렌즈 전 후보수`.
4. **합격 기준:** 기본 분해 렌즈의 평균 후보 감소율 **< 40%** (= "추리의 절반 이상은 여전히 플레이어 몫"). 교차점조차 보드 전체 후보 감소율 **< 60%**.
5. 불합격 시: 분해 렌즈를 "기본 보유"에서 **강등**하거나, 표시를 "칸 수 범위"에서 "단순히 빈칸 1개 존재 여부"로 더 낮춘다.

> **[비평 반영 — designHole #2 / mustFix #2 해결]** 분해 렌즈에서 "레벨 분해 후보 나열"을 완전 제거, "기여 칸 수 범위"로 정보량 하향. 이산 집합 자동 소거 문제 회피. MVP 0순위로 스프레드시트 정보이득 검증(합격선 후보감소율<40%)을 코드 전에 의무화.

### 6.4 solvable의 인간 가독성 강화 — 비평 #3 해결

> 비평 #3: "solvable = 논리적 유일해"는 "인간 작업기억 내 풀이 가능"을 보장하지 않는다. 13x10에서 정보 도구 없이 다중 연립을 손으로 푸는 건 비현실적 → 추측 클릭 도박 강제 → "운 사망은 계산 실수에서만"이 깨짐.

**v2 — solvable 정의를 "제약된 인간 추리 모델로 풀 수 있는 시드"로 강화:**

**Human-Solvable 생성기 제약 (확정 명세):**
1. **동시 연립 한계:** 어떤 칸의 정체도 **최대 3칸 동시 연립** 이내로 확정 가능해야 한다. 4칸 이상을 동시에 연립해야만 풀리는 시드는 **생성기가 거부(reject & regenerate)**.
2. **진입점 보장:** 보드에는 항상 **추론 없이 안전이 확정되는 "씨앗 칸" 최소 N개**(예: 모서리/저밀도 영역)가 존재. 첫 클릭이 도박이 되지 않게.
3. **추론 체인 깊이 제한:** "A를 풀어야 B, B를 풀어야 C" 식 체인이 **최대 깊이 4**를 넘지 않음. 그 이상은 거부.
4. **정보 최저선 보장:** **분해 렌즈는 시작 기본 보유(무작위 풀에서 제외)** → 정보 도구를 하나도 못 뽑는 최악의 런이 **존재하지 않음.**

**분해 렌즈 "기본 vs 무작위 풀" 모순 해소 (비평 #3 명시 요구):**
- v1: 분해 렌즈를 "기본"이라 하면서 7.1 MVP 아이템 16개에 포함 → 무작위 풀과 모순.
- **v2 확정:** 분해 렌즈는 **시작 시 가방에 고정 장착된 기본 아이템**이며, **드롭/상점 풀에 포함되지 않는다.** MVP 아이템 카탈로그(§5)는 분해 렌즈를 "기본(풀 제외)"으로 별도 표기. 정보 최저선이 **항상** 보장됨.

> **[비평 반영 — designHole #3 / mustFix #3 해결]** solvable을 "3칸 동시 연립·체인 깊이 4·씨앗 칸 보장"의 인간 가독성 모델로 강화. 분해 렌즈를 "기본 보유(풀 제외)"로 확정해 정보 최저선 보장 + v1의 기본/풀 모순 제거. → "정보 도구 전무 런 = 추측 도박" 시나리오 제거.

### 6.5 교차점 추리 (Cross-Point Deduction) — 빌드 깊이의 씨앗 (복잡도 우려 완화)

행 도구(측량 잣대)와 열 도구(가저의 눈)의 **교차 칸만** 최고 해상도가 된다. "어느 칸을 가장 선명하게 볼지"를 가방 배치로 선택 → **패킹 결정 = 추리 결정.**

> **복잡도 비평 완화 (complexityCreep #3 "두 좌표계 매핑 부담"):** 두 좌표계(가방/보드)의 정신적 매핑이 무겁다는 지적에 대해, **UI가 매핑을 대신 시각화**한다: 측량 잣대를 가방 row0에 놓으면, **보드 위 해당 커버 행에 은은한 가로 하이라이트 띠**가 표시된다(가방→보드 투영). 플레이어는 좌표를 머릿속으로 계산하지 않고 **하이라이트된 띠의 교차점**을 눈으로 본다. → 추상화 부담을 UI로 흡수.

### 6.6 난이도 슬라이더로서의 정보 경제

정보 유물을 많이 살수록 추리량↓(안전), 적게 살수록 맨몸 추리(스릴). 밸런싱 칼날을 플레이어 선택으로 분산. 단, 이것이 **밸런스 문제를 떠넘기는 것**이라는 비평(balanceRisk #4)에 대해 — **MVP 검증의 핵심 단일 질문**("분해 렌즈 기본값이 추리를 자동화하는가")은 §6.3의 스프레드시트로 **슬라이더와 무관하게 먼저 확정**한다. 슬라이더는 그 위에서의 *추가* 자율 조절일 뿐, MVP 재미 검증의 회피 수단이 아니다.

## 7. 시그니처 기믹 (악용 안전장치 포함)

### 7.1 환율이 아닌 "전리품 폭발" (Loot Burst) — 도파민의 심장 재정의

> v1의 "환율 펌프"는 §3에서 트랙 분리로 재정의됐다. 도파민 모먼트는 **생존 구걸(부활)이 아니라, 정확히 계산한 위험을 트랙 B 보상으로 폭발시키는 0.5초.**

- **메커니즘:** HP를 정확히 계산해 강칸을 처치 → 화면 흔들림 + 빨간 -L + **가방 무기들이 정렬 시너지로 골드·점수를 폭증**(트랙 B) → 점수판이 튀어오름.
- **안전장치:** 폭발은 **생존(트랙 A)에 닿지 않음** → "보상 폭발로 살아남기" 불가. 깡플레이 우회 차단.

### 7.2 레벨업 버스트 (Level-Up Burst) — 데스 스파이럴 방지 재설계 (비평 #6 해결)

> 비평 #6 + degenerate #1: v1의 "탐욕의 인장(HP 비율↓→보너스↑)" + "쥐어짜기 게이지(저잔량 연출)"가 **"HP 1 상주"를 모든 빌드의 지배전략**으로 만들어 **데스 스파이럴**(미믹 1회 즉사)을 직접 유발. 신중한 플레이를 처벌.

**v2 재설계 — 모든 "쥐어짜기" 보너스를 "HP 비율(잔량)"에서 "누적 소모량/레벨당 총 회수"로 전환:**

| 요소 | v1 (데스 스파이럴 유발) | v2 (안전) |
|---|---|---|
| **탐욕의 인장** | HP 1에서 EXP +50% (저잔량 상주 강제) | **삭제.** |
| **쥐어짜기 연출** | 저잔량일수록 화면 흔들림↑ (도박 미화) | **"이번 레벨에서 회수한 총 트랙B 점수"에 연동.** 신중하게 많이 모아도 동일하게 화려 |
| **레벨업 버스트 보너스** | — | 레벨업 시 **"직전 레벨 구간 누적 소모 HP"에 비례한 일회성 골드 보너스**(트랙 B). HP를 *많이 태웠으면* 보상 — 하지만 *언제 태웠든*(공격적이든 신중하든) 누적이 같으면 동일 보상 |

**정량 증명 (데스 스파이럴 제거):**
- v1: "HP 1 상주 플레이어" > "HP 5 유지 플레이어" (탐욕의 인장 보너스 차이) → 1 상주 강제.
- v2: 보너스가 "누적 소모 HP"에만 비례 → "HP를 5→0으로 한 번에 태운 신중한 플레이어"와 "HP를 1로 유지하며 야금야금 태운 도박 플레이어"가 **누적이 같으면 보상 동일.** → 저잔량 상주 유인 **소멸.** 신중한 계획 플레이가 동등하게 보상받음.

> **[비평 반영 — designHole #6 / degenerate #1 / mustFix #4 해결]** 탐욕의 인장 삭제. 모든 쥐어짜기 보너스를 "HP 비율"→"누적 소모량"으로 전환. 데스 스파이럴 유발 지배전략(HP 1 상주) 제거를 수치 증명.

### 7.3 연쇄 환전기 삭제 (비평 degenerate #3 해결)

> 비평: v1 "연쇄 환전기(같은 레벨 연속 처치 +10% 누적, 레벨업 시 리셋)"는 **"레벨업을 의도적으로 미루고 안전 약칸 연타"**를 유도 → 데스 스파이럴 + 추리 우회 노가다.

**v2 확정: 연쇄 환전기 삭제.** "레벨업 지연 보상" 메커니즘은 데스 스파이럴과 노가다를 동시에 키우므로 전면 배제. 트랙 B 증폭은 §5.7의 "지불 Lv 비례" 정적 산식으로만.

### 7.4 제로모션 연출 (#3 차용, 안전)

모션 애니메이션 철학적 배제. **카메라 shake + 숫자 트윈 + 화이트플래시**만으로 피드백 완결. 단 §7.2에 따라 연출 강도는 "저잔량"이 아니라 **"레벨당 회수 점수"**에 연동.

### 7.5 돌출 격리 슬롯 / 캠프 전용 패킹 / 빌드 태그 공유

- **돌출 격리 슬롯:** 비정형 가방 한 면에 저주/봉인 격리 → 실수 방지 + 저주 공간 역전.
- **캠프 전용 패킹:** §4.2의 3시점에서만. 즉발 템포 보존.
- **빌드 태그 + 스크린샷:** 클리어 화면에 가방 스크린샷 + 빌드 태그("정보 특화 / 골드 경제 / 여백 분산 / 실수방지형") → SNS 바이럴.

### 7.6 시그니처 모먼트 (재작성 — 트랙 분리 반영)

> HP가 4 남았다. 보드엔 Lv8 보라 슬라임 영역이 남았고, 한 칸만 더 안전 처치하면 이번 레벨이 끝난다. 미니 캠프에서 col4 한 열에 정렬해둔 **상인의 저울 + 계량추 + 숫돌**이 패시브로 빛난다. 옆 숫자 "9"가 **분해 렌즈로 "기여 칸 2~3개"**임을 알려준다 — 하지만 *어느* 칸이 *무슨* 레벨인지는 안 알려준다. 가저의 눈(col)과 측량 잣대(row)의 **교차 하이라이트 띠**가 만나는 칸이 "기여 칸"이라고 이진 표시된다. 거기에 인접한 다른 숫자 "5"와 도감(가고일은 쌍)을 연립한다 — 교차 칸은 **Lv4 가고일**, 그 짝은 대각선에. HP 4로 Lv4를 치면 HP 0 직전. **여기서 판단한다:** 지금 치면 죽지 않고, 계량추가 "Lv4×5%" 골드 보너스를 주며, 누적 소모 HP가 임계치라 레벨업 버스트 골드도 터진다. 클릭 — 화면이 흔들리고 빨간 -4와 초록 골드가 동시에 터진다. **트랙 A는 Vitality +4로 레벨업을 트리거**(화이트플래시, 완전회복, 최대HP+1) — 이건 *내가 지불한 4만큼* 정확히, 가방과 무관하게. **트랙 B는 저울+계량추+숫돌+버스트로 골드·점수가 폭증** — 이건 *내가 설계한 가방* 덕분. **"위험은 내가 푼 추리가 정확히 정산하고, 보상은 내가 둔 배치가 폭발시키는"** 0.5초. 살아남은 건 추리 덕분이지 가방 덕분이 아니다 — 가방은 그 위에 **자기효능감의 골드 분수**를 얹었을 뿐.

## 8. 진행 / 콘텐츠 구조

### 8.1 MVP (수직 슬라이스) — 3구역 단일 보드

> 비평 #4 해결을 위해 단일 보드를 **3구역으로 분할**해 한 판에 캠프 다회 발생.

- **단일 13x10 보드 1종** (Human-Solvable 생성기, §6.4).
- **3개 구역(Zone) 분할:** 보드를 좌/중/우 또는 외곽→중심 3단계로. 한 구역 안전 처치 완료 → **풀 캠프** 진입.
- **단일 통화 HP** + **이중 트랙 회수**(§3).
- **아이템 ~18개** (무기 4 / 방어구 4 / 유물 4(분해 렌즈는 기본·풀 제외) / 특수 4 + 예비 2).
- **시너지 축 2개 고정.**
- **캠프 3시점**(레벨업 미니캠프 / 구역 풀캠프 / 드래곤 직전).
- **클리어 화면 빌드 태그 + 스크린샷.**
- **MVP 0순위:** §6.3 분해 렌즈 정보이득 스프레드시트 검증 (코드 전).
- 목표 검증 질문: **"트랙 분리된 보상 폭발 + 칸수 기반 분해 렌즈 + 교차점 추리가 실제로 재밌고, 추리를 자동화하지 않는가."**
- **데이터화:** 아이템/몬스터/보드 규칙/레벨업 곡선 전부 **JSON** → 코드 수정 없이 튜닝.

### 8.2 1차 확장 — "강하 (Descent)" (MVP 검증 후에만)

- 원정: 3층 → (이후 7층). 한 층 = 점점 커지는 보드(9x7 → 13x10).
- 각 층 중앙에 드래곤알(미니보스) 공개. 처치 시 강하.
- 층 클리어: 가방 조형 + 아이템 3택 1. HP/레벨 유지, 보드 갱신.
- Descent Camp = §4.2 풀 캠프와 동일.

### 8.3 2차 확장 — 메타 진행 & 어센션

- **드래곤 비늘** 영구 통화(트랙 B 점수에서 환산): 새 캐릭터(시작 가방 형태 상이), 아이템 풀, 일일 챌린지 해금.
- **어센션 10단계 — 비평 #7 해결 (정보 빌드 표적 너프 → 전 빌드 균등):**

> 비평 #7 + balanceRisk #5: v1의 "어센션 스캔풀 제한(정보 아이템 풀만 제한)"은 **정보 빌드만 표적 너프** → 환율/골드 빌드는 영향 없음 → 아키타입 비대칭 → "한 판 더" 라벨링과 충돌.

  **v2 재설계 — 어센션은 정보 아이템 풀이 아니라 "보드 단서 밀도"를 줄인다:**
  - 고난도일수록 빈 타일(숫자 표시 칸) 비율↓, 초기 공개 영역↓, 추론 체인 깊이 상한↑.
  - → **모든 빌드가 균등하게 "머리를 더 써야"** 한다. 정보 빌드는 줄어든 단서를 보완하느라, 경제 빌드는 더 위험한 추론을 감수하느라 — **양쪽 다 동일하게 추리 부하 상승.**
  - 정보 아이템 풀은 건드리지 않음 → 아키타입 비대칭 제거.

- **4 아키타입 라벨링** (헌법 내, 시너지 축 2개 유지): **정보 특화 / 골드 경제 / 여백 분산 / 실수방지(방어구) 빌드.** 라벨만 부여해 빌드 정체성 강화.

> **[비평 반영 — designHole #7 / balanceRisk #5 해결]** 어센션을 "정보 풀 제한"→"보드 단서 밀도 감소"로 변경. 전 빌드 균등 추리 부하 상승 → 아키타입 비대칭 제거.

## 9. 밸런싱 철학 (엑셀로 조절할 변수 목록)

### 9.1 헌법 (절대 불변)

1. **단일 *지불* 통화 HP.** (회수는 트랙 A/B 2장부지만 지불 통화는 HP 하나.)
2. **시너지 축 2개 고정.** 곱연산·체인 영구 금지.
3. **위험 흡수·회복·부활 부재** + **트랙 A(생존) 가방 격리.**
4. **분해 렌즈는 "칸 수 범위"만** (분해 후보 나열 금지).

### 9.2 이중 안전망 — CI 필수 자동 검증

> 비평 designHole #8: v1의 "안전망 ②(깡플레이<5%)"는 "헌법 #1(흡수 부재로 깡플 원천차단)"과 **논리적 중복이자 모순**이다. 흡수가 불가능하면 깡플 성공률은 정의상 ~0이라 게이트가 무의미. 게이트가 의미를 가지려면 "환율 빌드가 레벨업 가속으로 깡플을 가능케 함"을 인정하는 셈 = designHole #1 재확인.

**v2 — 게이트를 재정의해 의미를 부여 (designHole #8 해결):**

- **안전망 ① — 정보이득 정량 측정 하베스 (MVP 0순위):** AI 추리 솔버로 각 유물 보유/미보유 시 **평균 후보 감소율**을 측정. §6.3 합격선(기본 렌즈 <40%, 교차점 <60%) 회귀 강제. 분해 렌즈가 추리를 자동화하면 빌드 타임에 적발.
- **안전망 ② — 트랙 분리 검증 게이트 (재정의):** v1의 "깡플레이<5%"를 **"트랙 A 불변성 검증"**으로 재정의. 자동 테스트가 확인하는 것:
  1. **동일 처치 시퀀스 → 빌드 무관 동일 레벨업 타이밍** (트랙 A가 가방에 오염되지 않음을 단언).
  2. **최대 트랙 B 빌드로도 "추론 없이 무작위 클릭" 클리어율 ~0** (= 트랙 분리가 실제로 생존을 차단함을 확인).
  - → 이 게이트는 헌법 #1을 **검증/반증하는 1차 지표.** 만약 ②-1이 깨지면(빌드별 레벨업 타이밍 차이 발생) §3의 트랙 분리가 코드에서 누수된 것 → 즉시 적발.

> **[비평 반영 — designHole #8 해결]** 깡플 게이트를 "트랙 A 불변성 + 무추론 클리어율 0" 검증으로 재정의. 헌법 #1과의 중복/모순을 제거하고, 게이트가 트랙 분리(designHole #1 해결책)의 **회귀 감시자**라는 명확한 의미를 부여.

## 10. Phaser 구현 노트

### 10.1 기술 스택 & 아키텍처 원칙

- **엔진:** Phaser 3 (TypeScript). `any` 지양, 함수 카멜케이스, try-catch 오류 처리 (사용자 코드 원칙 준수).
- **엔진 분리 (핵심):** Phaser는 **렌더링/입력/씬 전환만.** 게임 로직(보드 상태·Human-Solvable 시드 생성·트랙 A/B 회수 계산·시너지·분해 렌즈 정보 계산)은 **순수 TS 모듈**(`/core`)로 분리 → **Jest 단위 테스트** (안전망 ①②).

### 10.2 씬 구조

```
BootScene        에셋 로드, JSON 데이터 파싱
  └─ TitleScene  타이틀, 캐릭터/어센션 선택
       └─ GameScene        ★메인. 보드 Container + 가방 Container + HUD
            ├─ BoardLayer        13x10 동적 Container 그리드 (Tilemap 아님)
            ├─ BackpackLayer     5x5 비정형 Container, 좌표 스냅
            ├─ HudLayer          HP(트랙A) / 골드·점수(트랙B) 분리 표시
            ├─ CampOverlay       미니/풀 캠프 패널 (드래그&드롭·회전·구매)
            └─ FxLayer           카메라shake·숫자트윈·화이트플래시
       └─ ResultScene  클리어/사망. 빌드 태그 + 가방 스크린샷 캡처
```

### 10.3 핵심 데이터 모델 (순수 TS, 인터페이스 예시)

```typescript
// /core/types.ts — Phaser 비의존, Jest 검증 대상

interface MonsterDef {
  id: string;
  level: number;            // 고정 레벨 = 데미지 = 트랙A Vitality
  placementRule?: PlacementRule;  // 쥐→쥐왕열, 가고일 쌍 등
}

interface Cell {
  pos: { x: number; y: number };
  content: 'empty' | 'monster' | 'wall' | 'mimic' | 'mine';
  monsterId?: string;
  revealed: boolean;
  adjacentSum: number;      // 인접 8칸 레벨 총합 (= 표시 숫자)
}

interface PaymentResult {        // ★ 트랙 분리의 코드 표현 ★
  hpCost: number;                // 차감 HP (가방 개입 불가)
  trackA_vitality: number;       // = hpCost (1:1, 빌드 무관)
  trackB_gold: number;           // 가방 증폭 후 골드
  trackB_score: number;          // 가방 증폭 후 점수
}

interface BackpackItem {
  id: string;
  shape: GridShape;              // 테트로미노
  category: 'weapon' | 'defense' | 'relic' | 'special';
  synergyAxis: 'line' | 'void' | 'isolation' | 'none';
  // 효과는 trackB/정보에만 적용. trackA 수정 필드는 타입에 존재하지 않음(헌법 강제)
}

// 트랙 분리 핵심 함수 — 가방은 trackB·정보만 받고 trackA는 hpCost에 고정
function resolvePayment(
  monsterLevel: number,
  backpack: BackpackState
): PaymentResult {
  const hpCost = monsterLevel;
  return {
    hpCost,
    trackA_vitality: hpCost,                          // ★ 가방 미적용
    trackB_gold: applyBackpackGold(hpCost, backpack), // 가방 적용
    trackB_score: applyBackpackScore(hpCost, backpack),
  };
}
```

> **타입 시스템으로 헌법 강제:** `BackpackItem`에 `trackA`/`hp`/`heal`/`revive` 필드를 **아예 정의하지 않음** → 컴파일 타임에 "가방이 생존에 개입"하는 코드를 **타입 에러로 차단.** 헌법 #1을 코드 구조로 보증.

### 10.4 핵심 알고리즘 비용 (진짜 리스크)

엔진이 아니라 아래 3개가 일정 리스크:
1. **Human-Solvable 시드 생성기** (§6.4): 생성 후 "제약된 추리 솔버"로 풀어보고 3칸 연립·체인 깊이 4 위반 시 reject & regenerate. **MVP 핵심 난이도.**

3. **시너지 밸런싱** (트랙 B 산식): JSON 변수 회귀 테스트.

### 10.5 연출 트윈 (모션 애니 0)

### 11.2 남은 미해결 결정사항 (개발 중 데이터로 결정)

> 아래는 **설계로 닫지 못하고 플레이테스트/시뮬레이션 데이터로 확정해야 하는** 항목. 정직하게 미해결로 표기한다.

1. **[높음] 트랙 B의 "골드 vs 점수" 용도 분리 비율.** 골드(캠프 경제)와 점수(랭킹/메타)를 별도 풀로 둘지, 하나의 트랙 B를 양쪽에 쓸지. → MVP에서 "골드=캠프, 점수=결과화면"으로 시작하되, 메타 진행 도입 시 환산비를 데이터로 튜닝. **현 미결: 두 자원이 너무 분리되면 트랙 B 자체가 "이번 판 무의미"해져 동기 약화 위험.**
2. **[높음] 분해 렌즈 "칸 수 범위"가 너무 약해 가방이 장식이 될 위험.** §6.3 검증이 "<40%"를 통과하더라도, 반대로 "정보 가방을 깔 이유가 없을 만큼 약함"일 수 있음. **칼날의 양쪽 끝**(자동화 vs 무용)을 동시에 스프레드시트로 측정 필요. → 미결: 두 합격선(자동화 상한 + 유용성 하한) 사이 폭이 실재하는지는 시뮬레이션 전 불확정.
3. **[중간] 3구역 분할이 단일 보드의 "한 판 완결성"을 해칠 위험.** 구역 캠프가 너무 잦으면 즉발 추리 스프린트의 몰입이 끊김 (complexityCreep 우려의 변형). → 구역 2개 vs 3개를 플레이테스트로 결정.
4. **[중간] 인지 부하 총량.** 비평 complexityCreep의 핵심 — 캠프로 시간 분리해도 "보드 추리 + 가방 정렬 추적"의 인지 예산은 분리 안 됨. §6.5의 보드 하이라이트 투영 UI로 완화하나, **실측 전까지 "원작의 미니멀 우아함 훼손" 위험은 열려 있음.** → 신규 유저 온보딩 테스트 필수.
5. **[중간] 미믹·지뢰의 "계산 실수 외 사망" 경계.** Human-Solvable이 미믹/지뢰까지 "추론으로 회피 가능"을 보장하는지. 미믹이 "정보 도구로도 끝까지 불확정"이면 비평 #3의 도박이 재발. → 생성기가 미믹/지뢰도 추론 회피 가능하게 배치하는지 검증 필요.
6. **[낮음] 지뢰 절대선(Lv100)의 실제 필요성.** 트랙 분리로 깡플레이가 차단됐다면 지뢰 백스톱은 잉여일 수 있음. 단 안전망으로 보존(§9.2 게이트 ②-2가 0이 아니면 백스톱 작동). → 데이터로 잉여 확인 시 제거 검토.

### 11.3 잔존 철학적 긴장 (정직한 고지)

- **"흡수 부재 → 1차원 평면화" (balanceRisk):** 위험 흡수를 헌법으로 금지했으므로, 로그라이크의 고전적 "생존 vs 보상" 긴장이 없다. v2는 이를 **4카테고리(무기/방어구=실수방지/유물/특수)의 서로 다른 의사결정 축**으로 보완했으나, "방어구=실수 방지"가 "방패=생존"만큼 **감정적으로 묵직한 선택**인지는 미검증. → MVP의 핵심 재미가 "생존 긴장"이 아니라 **"추리 해상도 거래의 지적 긴장"**임을 받아들이는 장르적 베팅이다. 이 베팅이 50시간 수명을 지탱하는지가 최종 리스크.
- **복잡도 vs 우아함:** 비평이 옳다 — 원작은 "숫자 하나 + 클릭 하나"의 우아함이다. v2는 그 위에 가방을 얹어 **의도적으로 복잡도를 더했다.** 정당화는 "가방이 추리를 *깊게* 한다(교차점)"는 것이지만, **"두 번째 격자를 얹은 것이 순증 재미인가, 인지 과부하인가"**는 본질적으로 플레이테스트로만 답할 수 있는 열린 질문이다.

## 부록 A. 한 문단 최종 요약 (Executive Summary)

**백팩스위퍼**는 드래곤스위퍼의 "숫자 = 인접 8칸 몬스터 레벨의 총합" 추리를 100% 보존한 위에, 백팩히어로의 비정형 가방을 **"위험을 흡수하는 갑옷이 아니라 추리 해상도를 거래하고 보상을 증폭하는 작업대"**로 재해석한 연역형 격자 로그라이크다. v1의 치명적 자기모순(가방이 EXP를 증폭하면 레벨업=회복=생존이 증폭됨)을 **이중 트랙 모델**로 해결한다 — HP 지불은 항상 두 갈래로 회수되어, **트랙 A(Vitality, 지불 레벨에 1:1 비례, 가방 절대 불가침)**가 레벨업·완전회복·최대HP를 결정해 *생존은 빌드와 무관하게 결정론적*이고, **트랙 B(골드·점수, 가방이 증폭)**는 캠프 경제와 랭킹에만 쓰여 *보상은 가방이 폭발시키되 생존엔 닿지 못한다.* 분해 렌즈는 "분해 후보"를 나열하지 않고 "기여 칸 수 범위"만 주어(코드 전 스프레드시트로 후보 감소율<40% 검증) 이산 레벨 집합 자동 소거를 회피하며, Human-Solvable 생성기(3칸 동시 연립·체인 깊이 4·씨앗 칸 보장)가 정보 도구 없는 최악의 런에서도 추측 도박을 배제한다. 저잔량 데스 스파이럴(탐욕의 인장)은 삭제하고 모든 쥐어짜기 보너스를 "HP 비율"→"누적 소모량"으로, 노가다 유인은 "클릭 횟수"→"지불 Lv"로 묶어 신중한 플레이와 강칸 정확 추리를 동등·우선 보상한다. 캠프는 "레벨업 미니캠프 + 3구역 풀캠프"로 통일돼 한 판에 "패킹=추리" 빌드 깊이를 충분히 검증하고, 단일 통화·시너지 2축·흡수 부재·분해 렌즈 정보 상한의 4대 헌법과 정보이득 측정 + 트랙 A 불변성의 이중 CI 게이트로 1인 개발 밸런싱을 통제한다. 도파민의 심장은 "생존을 구걸하는 부활"이 아니라 **"위험은 내가 푼 추리가 정확히 정산하고, 보상은 내가 둔 가방 배치가 폭발시키는 0.5초의 자기효능감"**이다.
